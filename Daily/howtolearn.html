Processing is Awesome!

关注我微博的朋友之前会刷到我会发一些循环gif，评论常常也会问这是什么东西制作的，就是proceesing，一个基于java的编程语言。

未接触过编程的人，对编程的第一反应是枯燥无聊超难学。但当你接触processing之后，98%的人会问如何学？！

这篇文章就是以我这个之前零代码基础的小白学习processing的一些小经验，仅供大家参考。

我为什么选择processing？ 

太牛逼

图形 | 可视化 | 视觉

首先，我的工作是关于数据可视化。在接触编程之前，我只会Adobe Illustrator和Excel，所以想学一门编程语言去创造更多可视化效果。

当时犹豫选D3.js还是processing，业界更多用的是D3.js做可视化，毕竟它在数据可视化方面相对成熟，有很多模板可以生成很多图表；而processing做可视化，用我前同事老覃的话就是“没有做不出的图表！创造性超强！但…………就是全部要自己写。”（摊手）

直到我看到 @lisacrost 的这篇文章What I Learned Recreating One Chart Using 24 Tools这篇文章用同一组数据，用12种不同的软件和12种不同的编程去制作图表，就可视化的灵活性和学习的成本等做了比较。

你会看到，在Y轴上processing和D3.js（一款数据可视化功能丰富的库）都有很大的灵活性，但是processing位于y=0处，可以理解为它是一种学习起来并没有那么难，最后做到的效果却和d3.js旗鼓相当。果断选择processing。而且它在图形设计方面是强项，设计师如果学了这个的话，可以创作更多意想不到的作品。

所以喜欢可视化的设计师可以把它作为第一门学习的编程语言，当然正如processing官网所说的，它是一个灵活的视觉系编程语言，适合学生、艺术家、设计师、研究者和任何对processing感兴趣的人。

Processing is a flexible software sketchbook and a language for learning how to code within the context of the visual arts. Since 2001, Processing has promoted software literacy within the visual arts and visual literacy within technology. There are tens of thousands of students, artists, designers, researchers, and hobbyists who use Processing for learning and prototyping.

processing的第一组打卡（2017年2月10日）

这是我刚学不久之后做的效果。我做了第一张之后，改了一些参数，就可以生成第二三张效果。这种简单又极快反馈的编程语言强烈地吸引了我。

到后来的Coding365（畅想每天能做一张用processing生成的图），虽然以失败告终（捂脸），但也终究勉勉强强逼自己打了90天了。这个过程中确实会因为编程带来兴奋感、成就感、满足感。

如何开始学习？

  1. 入门学习：

    跟着processing官网上的example和tutorials学习，里面很多案例你可以在它的网页上改一些参数，马上可以看到效果，很快地去了解某一些代码的作用。

    官网也推荐了很多书籍，建议可以从创始人Casey Reas 和 Ben Fry写的其中一本书学，两本都可以买到中文翻译版本。第一本《Processing: A Programming Handbook for Visual Designers, Second Edition》（中文版叫《Processing语言权威指南》里面的小案例非常多，是一本工具书；第二本《Make: Getting Started with Processing, Second Edition》(这本书的第一版的中文版叫《爱上processing》，它也是我的入门书，写的简单有趣易学。

    如果你觉得看书枯燥的话，那可以跟着看画风很不一样的Daniel Shiffman的YouTube教程 The Coding Train

    学习。他录了很多课程，有不同的专题，包括processing入门、p5.js入门、Coding Challenge、machine learning等课程，常常你会发现你的学习进度都跟不上他的录课速度。

    2. processing有一个开源的社区叫OpenProcessing，上面有各种艺术家、设计师、编程爱好者等上传他们的代码，你可以看到他们写的东西，还可以fork他们的代码。看不懂写了啥？没关系，尝试改动他们的代码，会发现一些小惊喜的。

  2. 举一反三：
    1. 当你学会一个新的公式，或者技巧的时候，反复的在不同的练习中运用。这样你就回相对熟练的去用。

  3. 学会复用：
    1. 把一些常用的结构保存到一个文档里面，方便以后调用的时候查找。

  4. 更多输出：
    1. 在gif的基础上再做个输出，可以是网页/app

如果总结它的优势的话

  1. 适合零编程基础的人，学习曲线相对平缓。
  2. 反馈快，写完一个就可以看到生成的效果。
  3. 图形创造性强大，适合生成创意图形效果。
  4. 目前学习资源比较丰富，只要你能跨出墙。

学习中遇到困难怎么办？

  1. 忘记一些结构怎么写?
      可以在processing官网的reference去搜索写法，这个过程就像查字典一样。

  2. 遇到各种bug
      首先可以看控制栏中反馈的错误信息是什么。有些你可以直接fix，比如英文的分号，你输入了中文分号；有些不懂的话可以问google。 （google至少可以解决80%的问题）

在以上解决方案无果之后，可以私信一些比你厉害的人，注意说清楚你的问题。
